# Question 2

### 1. Define digital security risks and explain their impact on organizations.
<details>
  <summary>Show Answer</summary>
  Answer: Digital security risks refer to threats that can compromise the integrity, confidentiality, and availability of an organization's digital assets. These risks include Internet and network attacks, unauthorized access, hardware and software theft, information theft, and system failures.

  Impact on organizations:
  - Financial Loss: Due to data breaches and cyberattacks.
  - Reputation Damage: Loss of customer trust due to compromised security.
  - Legal Consequences: Non-compliance with data protection laws.
  - Operational Disruptions: Cyberattacks can shut down critical systems.
</details>

### 2. Discuss the importance of protecting personal data online.
<details>
  <summary>Show Answer</summary>
  Answer: Protecting personal data online is essential to prevent identity theft, financial fraud, and privacy invasion. Social media platforms and online services collect vast amounts of personal information. Without proper security measures, attackers can use this data for malicious activities.

  Preventive measures include:
  - Using strong, unique passwords and multi-factor authentication (MFA).
  - Avoiding clicking on suspicious links or emails (phishing attacks).
  - Regularly checking if accounts have been compromised using services like [Have I Been Pwned](https://haveibeenpwned.com/).
</details>

### 3. How does unauthorized access pose a threat to cybersecurity?
<details>
  <summary>Show Answer</summary>
  Answer: Unauthorized access refers to individuals gaining access to a system or data without permission.

  Threats include:
  - Data Theft: Hackers can steal sensitive company information.
  - System Manipulation: Attackers can modify or delete crucial files.
  - Identity Theft: Stolen personal data can be used for fraudulent activities.
  - Malware Installation: Hackers can introduce trojans, viruses, and spyware to compromise system integrity.
</details>

### 4. Explain the difference between hardware theft and software theft.
<details>
  <summary>Show Answer</summary>
  Answer:  
  - Hardware Theft: Involves stealing physical computer devices such as laptops, hard drives, and mobile phones. This can lead to data breaches if sensitive information is stored on the stolen device.  
  - Software Theft: Refers to piracy, unauthorized copying, or illegal distribution of licensed software. This results in financial losses for software developers and can also introduce malware risks if pirated software is used.
</details>

### 5. How do system failures contribute to security vulnerabilities?
<details>
  <summary>Show Answer</summary>
  Answer: System failures can expose a network to cyber threats by:
  - Disrupting security software, leaving systems unprotected.
  - Allowing unauthorized access if authentication mechanisms fail.
  - Losing critical data due to unplanned downtime.
  - Creating opportunities for cybercriminals to exploit unpatched vulnerabilities.
</details>

### 6. What is a digital security risk?
<details>
  <summary>Show Answer</summary>
  Answer: A digital security risk is the probability of a cyber threat exploiting a system's vulnerability to cause harm.  
  Common risks include:
  - Unauthorized access (hacking).
  - Data theft (stealing personal or corporate data).
  - Malware attacks (viruses, trojans, ransomware).
  - Phishing attacks (fraudulent attempts to obtain sensitive information).
</details>

### 7. Name two common types of cybersecurity threats.
<details>
  <summary>Show Answer</summary>
  Answer:  
  1. Phishing Attacks: Cybercriminals trick individuals into revealing sensitive information by posing as trustworthy entities (e.g., fake emails from banks or companies).  
  2. Ransomware Attacks: Malicious software encrypts the victim’s data and demands payment for its release.
</details>

### 8. How can users check if their data has been leaked online?
<details>
  <summary>Show Answer</summary>
  Answer: Users can check if their accounts have been compromised by visiting [Have I Been Pwned](https://haveibeenpwned.com/). This website allows individuals to enter their email addresses and check if their information has been exposed in data breaches.
</details>

### 9. What role does encryption play in securing data?
<details>
  <summary>Show Answer</summary>
  Answer: Encryption is a method of converting data into a coded format to prevent unauthorized access.
  - Protects sensitive information from hackers.
  - Ensures secure communication over the internet (e.g., HTTPS).
  - Prevents data leaks if devices are stolen or compromised.
  - Common encryption algorithms include AES (Advanced Encryption Standard) and RSA (Rivest-Shamir-Adleman).
</details>

### 10. How can companies protect themselves from cyber threats?
<details>
  <summary>Show Answer</summary>
  Answer:
  1. Implement Strong Password Policies: Require employees to use complex passwords and change them regularly.
  2. Use Multi-Factor Authentication (MFA): Adds an extra layer of security.
  3. Install Firewalls and Antivirus Software: Helps detect and prevent cyber threats.
  4. Train Employees on Cybersecurity Awareness: Educating staff about phishing and social engineering attacks.
  5. Regular Software Updates and Patching: Fix security vulnerabilities in operating systems and applications.
</details>

# Questions 3

## Q1: Define spoofing attacks and explain their impact on cybersecurity.
<details>
  <summary>Show Answer</summary>
  A spoofing attack is an attempt to breach a network’s security by altering the source
  addresses of packets, making them appear as though they came from a trusted source rather than
  an untrusted attacker. These attacks can lead to unauthorized access, data theft, and identity fraud.
</details>

---

## Q2: Discuss different types of spoofing attacks and provide examples.
<details>
  <summary>Show Answer</summary>
  - IP Spoofing: The attacker modifies the source IP to bypass security filters.
  - MAC Spoofing: Faking a device’s MAC address to access restricted networks.
  - Email Spoofing: Sending fraudulent emails that appear to come from a legitimate sender.
  - DHCP Spoofing: Manipulating DHCP responses to control network traffic.
</details>

---

## Q3: How can organizations defend against IP and email spoofing attacks?
<details>
  <summary>Show Answer</summary>
  - Use Access Control Lists (ACLs) to filter incoming traffic.
  - Implement authentication mechanisms like SPF, DKIM, and DMARC for emails.
  - Enable intrusion detection systems (IDS) to monitor spoofing attempts.
  - Train employees to recognize spoofed emails.
</details>

---

## Q4: What is a hijacking attack, and how does it compromise security?
<details>
  <summary>Show Answer</summary>
  A hijacking attack occurs when an attacker takes control of a communication session
  between two parties, disconnects one user, and impersonates them. This can result in data theft,
  unauthorized transactions, and identity fraud.
</details>

---

## Q5: Explain the role of authentication in preventing spoofing attacks.
<details>
  <summary>Show Answer</summary>
  Authentication verifies the identity of users or devices before granting access.
  Techniques like multi-factor authentication (MFA), digital certificates, and encrypted
  communications help prevent spoofing attacks.
</details>

---

## Q6: What is an IP spoofing attack?
<details>
  <summary>Show Answer</summary>
  An IP spoofing attack involves falsifying the source IP address in network packets to
  bypass security filters and appear as a trusted user within the network.
</details>

---

## Q7: Name two common spoofing techniques used by hackers.
<details>
  <summary>Show Answer</summary>
  - Man-in-the-Middle (MITM) Attack: Intercepting and modifying communication between two parties.
  - Email Spoofing: Sending fake emails that appear legitimate.
</details>

---

## Q8: How does session hijacking work?
<details>
  <summary>Show Answer</summary>
  In session hijacking, an attacker steals the user’s session token or ID to gain
  unauthorized access to an account or system.
</details>

---

<details>

## Q11: Define password attacks and their role in cybersecurity breaches.
<details>
  <summary>Show Answer</summary>
  A password attack is an attempt to gain unauthorized access by cracking or stealing a

## Q12: Explain different types of password attacks with examples.
<details>

<details>

# Question 4

### 1. Define a computer network and explain its main benefits.  

<details>  
  <summary>Show Answer</summary>  
  A computer network is a collection of computers, devices, and peripherals connected together to share resources.  

  Benefits include:  
  - File sharing for easy collaboration.  
  - Device sharing, such as printers and storage devices.  
  - Software sharing using multi-user licenses.  
  - Voice and video calls over the network.  
  - Shared Internet access for efficient connectivity.  
</details>  

### 2. What is the difference between a MAC address and an IP address?  

<details>  
  <summary>Show Answer</summary>  
  - MAC Address: A physical, unique identifier assigned to a network interface card (NIC). It does not change and is used within a local network.  
  - IP Address: A logical, changeable address assigned to a device for communication over a network. It helps route data between networks.  
</details>  

### 3. Explain the classification of networks based on geographical area.  

<details>  
  <summary>Show Answer</summary>  
  - PAN (Personal Area Network): Covers a small area, like a single user’s workspace.  
  - LAN (Local Area Network): Covers a limited area, such as an office or school.  
  - MAN (Metropolitan Area Network): Covers a city-wide area, connecting multiple LANs.  
  - WAN (Wide Area Network): Covers large areas, such as countries or continents.  
</details>  

### 4. Describe the advantages and disadvantages of the bus topology.  

<details>  
  <summary>Show Answer</summary>  
  Advantages:  
  - Simple and easy to set up.  
  - Requires minimal cables, making it cost-effective.  
  - Reliable for small networks.  

  Disadvantages:  
  - A single cable failure can disrupt the entire network.  
  - Difficult to troubleshoot issues.  
  - Slower performance during peak traffic.  
</details>  

### 5. Compare the star and mesh topologies.  

<details>  
  <summary>Show Answer</summary>  
  - Star Topology: All devices connect to a central hub or switch. It is easy to expand and troubleshoot but fails if the central device stops working.  
  - Mesh Topology: Each device connects to multiple other devices. It provides high fault tolerance and security but is expensive due to excessive cabling.  
</details>  

### 6. What is the OSI model, and why is it important?  

<details>  
  <summary>Show Answer</summary>  
  The OSI (Open Systems Interconnection) model is a framework that standardizes network communication into seven layers:  

  1. Physical – Transmits raw bits.  
  2. Data Link – Ensures error-free data transfer.  
  3. Network – Handles IP addressing and routing.  
  4. Transport – Provides end-to-end communication.  
  5. Session – Manages sessions between devices.  
  6. Presentation – Converts and encrypts data.  
  7. Application – Interfaces with end users.  

  It helps different systems communicate and simplifies troubleshooting.  
</details>  

### 7. Explain the function of the Transport Layer in the OSI model.  

<details>  
  <summary>Show Answer</summary>  
  The Transport Layer is responsible for:  
  - Dividing data into segments.  
  - Providing end-to-end communication between devices.  
  - Ensuring error recovery and flow control.  
  - Using protocols like TCP (Transmission Control Protocol) and UDP (User Datagram Protocol).  
</details>  

### 8. What is the function of a router, and how does it differ from a switch?  

<details>  
  <summary>Show Answer</summary>  
  - Router: Connects different networks, directs data between them, and assigns IP addresses.  
  - Switch: Operates within a single network and directs data between devices.  

  Routers manage network traffic across the Internet, while switches work within LANs.  
</details>  

### 9. Describe the concept of data encapsulation in networking.  

<details>  
  <summary>Show Answer</summary>  
  Data encapsulation is the process of adding headers and trailers to data as it moves through the OSI layers. It helps organize data for transmission, ensuring it reaches the correct destination with necessary information like source and destination IP addresses.  
</details>  

### 10. What are network protocols, and why are they necessary?  

<details>  
  <summary>Show Answer</summary>  
  Network protocols are rules that define how devices communicate.  

  Examples include:  
  - HTTP (Hypertext Transfer Protocol) – For web browsing.  
  - FTP (File Transfer Protocol) – For file transfers.  
  - SMTP (Simple Mail Transfer Protocol) – For sending emails.  

  Protocols ensure interoperability and standardize communication between devices.  
</details>
